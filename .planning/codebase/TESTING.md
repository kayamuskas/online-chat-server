# Testing Patterns

**Analysis Date:** 2026-04-20 (updated — reflects Phase 6.1 implementation)

## Test Framework

### Unit / Integration Tests (API)

**Runner:** Jest (via `ts-jest`)
**Config:** `apps/api/jest.config.ts`
**Run command:** `pnpm --filter @chat/api test`

### E2E / Browser Tests

**Runner:** Playwright (`@playwright/test`)
**Config:** `playwright.config.ts` (repo root)
**Run command:** `pnpm test:e2e`
**Browsers:** Chromium (Desktop Chrome profile), headless

> **Policy:** Playwright is the **sole** browser automation framework for all E2E and UAT tests.
> Cypress, Selenium, and other browser automation tools are out of scope (see TEST-01 in REQUIREMENTS.md).

## Test File Organization

### API unit/integration tests

```
apps/api/src/__tests__/
  auth/          — register, login, logout, session, password-reset
  messages/      — messages-transport (gateway fanout)
  presence/      — presence.gateway
  rooms/         — rooms-controller
  contacts/      — contacts-controller
```

Naming: `*.spec.ts` co-located under `__tests__/` mirroring the source module tree.

### E2E Playwright tests

```
e2e/
  global-setup.ts          — Creates alice+bob fixtures, saves storageState files
  helpers/
    api-setup.ts            — REST helpers: createAndSignIn, createRoom, joinRoom, setupFriendship, openDm
    ui-helpers.ts           — Playwright page helpers: signInViaUi, openRoomChat, sendMessage, waitForMessage
  realtime/
    ws-auth.spec.ts         — UAT #1: Authenticated WS handshake
    room-realtime.spec.ts   — UAT #2/#3: Room message delivery and edit fanout
    dm-realtime.spec.ts     — UAT #4/#5: DM delivery and edit fanout
    presence.spec.ts        — UAT #6: Presence request-response
    autoscroll.spec.ts      — UAT #7: Smart autoscroll and new-message pill
    reconnect.spec.ts       — UAT #8: Reconnect recovery and history catch-up
```

Session state files (generated by global-setup, gitignored):
- `.alice-session.json` — Playwright storageState for alice
- `.bob-session.json`   — Playwright storageState for bob
- `.e2e-fixtures.json`  — User IDs, room ID, DM ID

## Test Structure

### Jest (API)

```typescript
describe('FeatureName', () => {
  let service: FeatureService;

  beforeEach(async () => {
    const module = await Test.createTestingModule({ ... }).compile();
    service = module.get(FeatureService);
  });

  it('should do X', async () => {
    expect(await service.doX()).toBe(expected);
  });
});
```

### Playwright (E2E)

```typescript
test.describe('UAT #N — Feature', () => {
  test('scenario description', async ({ browser }) => {
    const ctx = await browser.newContext({ storageState: '.alice-session.json' });
    const page = await ctx.newPage();
    try {
      await page.goto('/');
      await page.waitForSelector('.app-layout', { timeout: 8_000 });
      // ... test steps
    } finally {
      await ctx.close();
    }
  });
});
```

## Mocking

### Jest (API)

**Framework:** Jest manual mocks + `@nestjs/testing`

**What to mock:**
- `PostgresService` — replaced with in-memory or mock query implementations
- `BullMQ` queues — mocked at module boundary
- External mail service — `MailService` stubbed; SMTP never required

**What NOT to mock:**
- Business logic and domain rules — these must be exercised with real service instances
- `SESSION_COOKIE_NAME` constant — imported directly in both WS auth and middleware

### Playwright (E2E)

No mocking. Tests run against the full live stack (API + DB + Redis).
- `global-setup.ts` provisions real users/rooms/DMs via the live API before tests run.
- `storageState` is used to skip UI re-login per test and avoid rate-limiting.

## Fixtures and Factories

### API (Jest)

Test data is constructed inline per spec file. No shared factory library exists yet.

### E2E (Playwright)

**Shared fixtures:** `e2e/global-setup.ts` creates one alice + one bob user per test run:
- alice and bob are friends
- both are members of a shared room
- a DM conversation is open between them

Fixture data written to `.e2e-fixtures.json`, imported by specs as:
```typescript
import fixtures from '../../.e2e-fixtures.json' assert { type: 'json' };
const fx = fixtures as E2EFixtures;
```

## Common Patterns

### Async Testing (Jest)

```typescript
it('should resolve', async () => {
  await expect(service.asyncOp()).resolves.toEqual({ ... });
});
```

### Async Testing (Playwright)

```typescript
await page.waitForSelector('.app-layout', { timeout: 8_000 });
await expect(page.locator('.msg-bubble__content', { hasText: text })).toBeVisible();
```

### storageState sessions (Playwright)

Global setup saves authenticated browser sessions to avoid UI sign-in in every test.
Sessions are post-processed to strip `secure: true` so cookies work over `http://localhost`:

```typescript
// global-setup.ts
await ctx.storageState({ path: '.alice-session.json' });
stripSecureFlag('.alice-session.json'); // sets secure:false on all cookies
```

## Current Validation Workflow

1. Start the full stack: `docker compose up` or local dev servers
2. Run unit/integration tests: `pnpm --filter @chat/api test`
3. Run E2E suite: `pnpm test:e2e`
   - Requires web at `http://localhost:4173` and API at `http://localhost:3000`

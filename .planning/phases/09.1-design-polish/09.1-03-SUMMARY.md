# Phase 9.1 Plan 03: Frozen DM Redesign and Ghost Contact Display Summary

Replaced the minimal "read-only" badge in DmChatView with a full frozen-banner block (banned and account_deleted variants), added DM header action buttons, disabled composer states, and "[deleted]" italic styling for ghost contacts in ContactsSidebar.

## Tasks Completed

| # | Name | Commit | Files |
|---|------|--------|-------|
| 1 | Frozen DM banner, header buttons, disabled composer | fb3f177 | DmChatView.tsx, styles.css |
| 2 | Ghost contact display in ContactsSidebar | 0f7f02c | ContactsSidebar.tsx, styles.css |

## What Was Built

### Task 1: DmChatView redesign (fb3f177)

- Added `frozenReason` and `frozenAt` state, populated from `result.conversation.frozen_reason` and `result.conversation.frozen_at` in both the eligible and ineligible branches of `openConversation()`
- Added `onUnbanUser` and `onBanUser` optional callback props to `DmChatViewProps`
- Replaced the `<span className="rooms-badge rooms-badge--private">read-only</span>` with three-way header render:
  - `account_deleted`: header shows username + "deleted" badge
  - `banned` (frozen, not account_deleted): header shows username + "View profile" (disabled) + "Unban user" buttons
  - Normal (not frozen): header shows username + "View profile" (disabled) + "Ban user" (danger) buttons
- Added `frozen-banner` block after `<MessageTimeline>` inside `.rooms-view__body--scroll`:
  - Label: "⊘ ACCOUNT DELETED" or "⊘ CONVERSATION FROZEN"
  - Optional freeze date formatted as long-form date
  - Explanatory text variant by reason
  - "Unban to resume" button only for banned variant when `onUnbanUser` is provided
- Replaced always-rendered `<MessageComposer readOnly={frozen}>` with three-way composer render:
  - `account_deleted`: disabled div with "This account has been deleted"
  - `banned`: disabled div with "Messaging disabled — user is banned"
  - Normal: full `<MessageComposer>` as before

CSS additions:
  - `.frozen-banner` — warm yellow (#fbf4e4) background, dashed #e7d3ae border, centered text block
  - `.frozen-banner__label` — monospace, 600 weight, 0.1em letter-spacing
  - `.frozen-banner__date` — monospace, small, muted
  - `.frozen-banner__text` — small body text
  - `.frozen-banner__link` — borderless button, accent color, underlined
  - `.rooms-view__composer--disabled` — flex center, border-top, muted background
  - `.composer-disabled-text` — monospace, small, muted
  - `.rooms-view__header-actions` — flex row, gap 6px, margin-left auto

### Task 2: Ghost contact display (0f7f02c)

- In `ContactsSidebar.tsx`, the `contacts-sidebar__name` span conditionally appends `contacts-sidebar__name--deleted` when `c.username === '[deleted]'`
- CSS: `.contacts-sidebar__name--deleted { font-style: italic; color: var(--color-muted, #888); }`
- Primarily defensive styling — per plan notes, ghost contacts (where friendship row is cleaned up per D-9.1-15) won't appear in the friends sidebar normally; the primary UX is the ACCOUNT DELETED banner in DmChatView

## Deviations from Plan

### Auto-fixed Issues

None — plan executed as written with one minor adaptation:

**CSS variable name alignment:** Plan referenced `--mono`, `--ink`, `--muted`, `--muted-2`, `--accent`, `--line`, `--stripe` but the project's `:root` uses `--font-mono`, `--color-text`, `--color-muted`, `--color-accent`, `--color-border`, `--color-bg`. Used the actual project tokens with hardcoded fallbacks, matching the pattern used throughout the existing stylesheet.

## Known Stubs

None — all frozen DM UI is wired to real `frozen_reason` / `frozen_at` data from the API. The `onUnbanUser` and `onBanUser` props are optional callbacks; callers (App.tsx) will wire them in a subsequent plan if not already done.

## Threat Flags

None — this plan adds no new network endpoints, auth paths, or data access patterns. It is purely a rendering layer change consuming existing API data.

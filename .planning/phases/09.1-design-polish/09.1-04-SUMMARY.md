# Phase 9.1 Plan 04: Attachment Card Rendering and Timeline Divider Restyling Summary

**One-liner:** Replaced plain attachment links with styled file-type-icon cards and restyled both timeline dividers (older messages, new unread) to the chat-divider dashed-line pattern.

## What Was Built

### Task 1 — Attachment Cards (D-9.1-16)

Added two pure helper functions before the component:

- `getFileTypeIcon(filename)` — maps 20+ file extensions to short labels: DOC, PDF, TXT, IMG, ZIP, AUD, VID, XLS, CSV, FILE
- `formatFileSize(bytes)` — formats bytes as B / KB / MB

Replaced the plain `<a className="msg-attachment-link">` with a `.attachment` card layout:

```
[ icon box ] [ filename         ] [ Download ]
             [ size · comment   ]
```

Added CSS: `.attachment`, `.attachment__icon`, `.attachment__info`, `.attachment__name`, `.attachment__meta`.

### Task 2 — Timeline Dividers (D-9.1-17, D-9.1-18)

**Older messages divider:** Changed `<button className="msg-timeline__history-status__btn">` to `<button className="chat-divider chat-divider--clickable">` with label "older messages ↑". The `onClick` handler invoking `onLoadOlder` is preserved exactly.

**Unread divider:** Changed `<li className="msg-timeline__divider"><span>new ↓</span></li>` to `<li className="chat-divider">new ↓</li>`.

Added CSS for `.chat-divider` (display:flex, dashed lines via ::before/::after pseudos, monospace 11px) and `.chat-divider--clickable` (cursor:pointer, hover accent color). Removed the now-redundant `.msg-timeline__history-status__btn` CSS rule block.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] React fragment required for sibling JSX in return**
- **Found during:** Task 2
- **Issue:** Placing `{popoverTarget && ...}` after `</table>` inside the existing `return (<table>...)` caused TS error TS1005 (')' expected) тАФ the table was the sole root element
- **Fix:** Wrapped `<table>` and the popover expression in a `<>...</>` React fragment
- **Files modified:** apps/web/src/features/rooms/RoomMembersTable.tsx
- **Commit:** c6a292e

**2. [Rule 2 - Convention] CSS variable names use project conventions**
- **Found during:** Task 1
- **Issue:** Plan specified `var(--mono)`, `var(--accent)`, `var(--ink)` from design reference, but project's `styles.css` uses `var(--font-mono)`, `var(--color-accent)`, `var(--color-text)` with appropriate fallbacks
- **Fix:** Used project-standard CSS variable names with hardcoded fallbacks matching the design intent

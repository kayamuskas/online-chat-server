```typescript
function parseSendRequestBody(body: unknown): { targetUsername: string; message?: string } {
  const b = (body ?? {}) as { targetUsername?: unknown; message?: unknown };
  if (typeof b.targetUsername !== 'string' || b.targetUsername.trim().length === 0) {
    throw new BadRequestException('targetUsername is required');
  }
  return {
    targetUsername: b.targetUsername.trim(),
    message: typeof b.message === 'string' ? b.message.trim() || undefined : undefined,
  };
}

function parseBanBody(body: unknown): { targetUserId: string } {
  const b = (body ?? {}) as { targetUserId?: unknown };
  if (typeof b.targetUserId !== 'string' || b.targetUserId.trim().length === 0) {
    throw new BadRequestException('targetUserId is required');
  }
  return { targetUserId: b.targetUserId.trim() };
}
```

Implement these 12 routes in this exact order (order matters for NestJS route resolution —
static segments before parameterized segments):

```
GET    requests/outgoing           → getOutgoingRequests  200
POST   requests                    → sendFriendRequest    201 Created
GET    requests                    → getIncomingRequests  200
POST   requests/:id/accept         → acceptRequest        200
POST   requests/:id/decline        → declineRequest       204 No Content
DELETE requests/:id                → cancelRequest        204 No Content
GET    friends                     → getFriends           200
DELETE friends/:userId             → removeFriend         204 No Content
POST   bans                        → banUser              204 No Content
GET    bans                        → getMyBans            200
DELETE bans/:userId                → unbanUser            204 No Content
POST   dm/:userId                  → initiateDm           200
```

Response body shapes:
- GET /requests → return `{ requests }` (array of IncomingFriendRequestView)
- GET /requests/outgoing → return `{ requests }` (array of FriendRequest)
- POST /requests → return `{ request }` (FriendRequest)
- POST /requests/:id/accept → return `{ friendship }` (Friendship)
- GET /friends → return `{ friends }` (array of FriendWithPresence)
- GET /bans → return `{ bans }` (array of UserBan)
- POST /dm/:userId → return `{ conversation, eligible }` from contactsService.initiateDm()
- All 204 handlers return void with no body

Declare `@Get('requests/outgoing')` BEFORE `@Post('requests/:id/accept')` in the file to
prevent NestJS treating "outgoing" as an :id parameter.

## Component Inventory

### New element: "↓ новые сообщения" indicator

This is the only net-new UI element in Phase 06.1.

**Anatomy:**
```
.msg-timeline__new-messages-btn
  └── "↓ новые сообщения"   (button text, Unicode down-arrow + text)
```

**Positioning:** Absolutely positioned inside `.msg-timeline` (the scroll container's parent), centered horizontally, anchored `bottom: 16px` above the message composer. Uses `position: absolute; bottom: 16px; left: 50%; transform: translateX(-50%); z-index: 10`.

**Visual spec:**
- Background: var(--color-accent) = #5b7cf6
- Text color: #ffffff
- Font-size: 13px (0.8125rem), font-weight: 600
- Padding: 8px 16px (top/bottom 8px, left/right 16px)
- Border-radius: var(--radius) = 0.375rem (matches existing pill patterns)
- Min-height: 44px (accessibility touch target)
- Cursor: pointer
- box-shadow: 0 2px 8px rgba(0,0,0,0.45) (elevation to float above timeline content)
- Transition: opacity 0.15s, transform 0.15s

**States:**
- Hidden: `display: none` (or `opacity: 0; pointer-events: none` for animation)
- Visible: appears when `isScrolledUp === true` AND at least one new real-time message has arrived
- Hover: `opacity: 0.88` (matches existing `.btn:hover:not(:disabled)` pattern)
- Active/click: scrolls timeline to bottom; hides indicator immediately

**Show / hide trigger:**
- Show when: user has scrolled more than 100px from the bottom AND a `message-created` WS event delivers a new message
- Hide when: user scrolls back to within 100px of the bottom (or clicks the button)

**Accessibility:**
- `role="button"` (it is a `<button type="button">`)
- `aria-label="Scroll to new messages"` (English aria-label for screen readers; visible text is Russian)
- Must be focusable and operable via Enter/Space

### Modified element: MessageTimeline scroll container

The existing `.msg-timeline` `<div>` gains:
- `position: relative` (to anchor the indicator)
- Scroll event listener to compute `isScrolledUp`
- New prop `onNewMessagesClick` (optional callback, called by the indicator button)
- New prop `hasNewMessages: boolean` (indicator visibility signal from parent)

The scroll container itself (`msg-timeline__list` parent or the `.msg-timeline` root) must be the element with `overflow-y: auto` and a defined height. Confirm the scroll measurement uses `scrollHeight - scrollTop - clientHeight > 100` as the "scrolled up" threshold.

### Unmodified elements (wiring only, no visual change)

These components receive new props or event subscriptions but their rendered output does not change visually:

| Component | Change type |
|-----------|-------------|
| RoomChatView | Subscribes to message-created, message-edited events; emits joinRoom/leaveRoom on mount/unmount |
| DmChatView | Same as RoomChatView for joinDm/leaveDm |
| ContactsSidebar | Calls getPresence on mount and on a timed poll interval; passes status to PresenceDot |
| PresenceDot | No change — already renders online/afk/offline states correctly |
| App.tsx | Wraps authenticated shell in SocketProvider; connect on auth, disconnect on sign-out |

## Interaction Contracts

### Autoscroll rules (D-05)

1. When a new message arrives via WS AND `isScrolledUp === false`: scroll the timeline to bottom immediately (no animation, or smooth scroll at discretion of executor — smooth scroll is acceptable if it does not cause jitter).
2. When a new message arrives via WS AND `isScrolledUp === true`: do NOT scroll. Show the "↓ новые сообщения" indicator.
3. "Scrolled up" threshold: `scrollHeight - scrollTop - clientHeight > 100` pixels.
4. When user manually scrolls to within 100px of bottom: hide the indicator automatically.
5. When user clicks the indicator: scroll to bottom (smooth), hide indicator.

### Socket lifecycle (D-04)

1. Socket connects after `handleAuthenticated` fires in App.tsx.
2. Socket disconnects on `handleSignedOut`.
3. On entering a room conversation: emit `joinRoom` with `{ roomId }`.
4. On leaving a room conversation (unmount or navigation away): emit `leaveRoom` with `{ roomId }`.
5. On entering a DM conversation: emit `joinDm` with `{ conversationId }`.
6. On leaving a DM conversation: emit `leaveDm` with `{ conversationId }`.
7. Incoming `message-created` payload: append to messages list; trigger autoscroll logic.
8. Incoming `message-edited` payload: update the matching message in the list by ID.

### Reconnect recovery (D-06)

1. On socket `reconnect` event: silently re-emit `joinRoom` or `joinDm` for the active conversation (if any).
2. Fetch missed messages: `GET /api/v1/messages/room/:roomId/history?afterWatermark={lastWatermark}` (or DM equivalent).
3. Merge fetched messages into the list without duplicates (use message ID as dedup key).
4. No user-visible spinner, no toast, no banner.
5. Debounce reconnect refetch: wait 500ms after reconnect event fires before issuing REST call (avoid hammering on flapping connections; one pending fetch at a time).

### Presence polling (D-03)

1. ContactsSidebar calls `getPresence` via WS (request/response) on component mount.
2. Polling interval: every 30 seconds while the sidebar is mounted.
3. Each poll response updates the relevant contact's status in local state.
4. No loading indicator for presence polls — silent background update.

## Pattern Assignments

### `apps/web/src/lib/socket.ts` (utility, event-driven)

**Analog:** `apps/web/src/lib/api.ts`

**Imports pattern** (`api.ts` lines 41–43):
```typescript
import { SERVICE_PORTS } from "@chat/shared";

const BASE_URL = `http://localhost:${SERVICE_PORTS.apiHttp}/api/v1`;
```
Apply the same `SERVICE_PORTS` import — socket URL uses `SERVICE_PORTS.apiHttp` (3000).

**Core singleton pattern** — new file, modelled after `api.ts` module-scope state. The entire `api.ts` is a module-level singleton (no class, no export default, just exported functions sharing `BASE_URL` state). Follow that same style:

```typescript
// apps/web/src/lib/socket.ts
import { io, Socket } from "socket.io-client";
import { SERVICE_PORTS } from "@chat/shared";

let socket: Socket | null = null;

export function connectSocket(): Socket {
  if (socket?.connected) return socket;
  socket = io(`http://localhost:${SERVICE_PORTS.apiHttp}`, {
    withCredentials: true,   // session cookie "session" — verified ws-auth.ts
    autoConnect: true,
  });
  return socket;
}

export function getSocket(): Socket | null {
  return socket;
}

export function disconnectSocket(): void {
  socket?.disconnect();
  socket = null;
}
```

**Key constraint:** `withCredentials: true` is mandatory — backend `ws-auth.ts` reads the `session` cookie from the WS handshake. Without it the server disconnects the socket immediately.

### `apps/web/src/features/socket/SocketProvider.tsx` (provider, event-driven)

**Analog:** No direct React Context analog exists. Closest behavioral pattern: `App.tsx` auth lifecycle (`handleAuthenticated` / `handleSignedOut` on lines 108–119) and `useEffect` with `user` dep (lines 287–300).

**Auth lifecycle pattern from `App.tsx`** (lines 108–119, 287–300):
```typescript
function handleAuthenticated(nextUser: PublicUser) {
  setUser(nextUser);
  // side-effect on auth success
}

function handleSignedOut() {
  setUser(null);
  // cleanup on sign-out
}

useEffect(() => {
  if (!user) {
    // reset all derived state
    return;
  }
  // load data that depends on auth
}, [user, loadContacts, loadPendingRequests]);
```

**Core provider pattern** — new, no existing Context in codebase. Use `useRef` (not `useState`) for the socket ref to avoid re-rendering the whole tree:

```typescript
// apps/web/src/features/socket/SocketProvider.tsx
import { createContext, useContext, useEffect, useRef } from "react";
import { connectSocket, disconnectSocket } from "../../lib/socket";
import type { Socket } from "socket.io-client";

const SocketContext = createContext<Socket | null>(null);

export function SocketProvider({
  authenticated,
  children,
}: {
  authenticated: boolean;
  children: React.ReactNode;
}) {
  const socketRef = useRef<Socket | null>(null);

  useEffect(() => {
    if (authenticated) {
      socketRef.current = connectSocket();
    } else {
      disconnectSocket();
      socketRef.current = null;
    }
  }, [authenticated]);

  return (
    <SocketContext.Provider value={socketRef.current}>
      {children}
    </SocketContext.Provider>
  );
}

export function useSocket(): Socket | null {
  return useContext(SocketContext);
}
```

**CRITICAL anti-pattern to avoid:** `useState<Socket | null>` would trigger a full re-render on connect/disconnect. Use `useRef`.

**WS subscription useEffect to add** (copy pattern from RESEARCH.md Pattern 3):
```typescript
useEffect(() => {
  if (!socket || !roomId) return;

  socket.emit("joinRoom", { roomId });

  // Named handlers required for .off() cleanup
  function onMessageCreated(payload: unknown) {
    const msg = mapPayloadToView(payload);   // same mapMessageView from api.ts
    setMessages(prev => {
      if (prev.some(m => m.id === msg.id)) return prev;  // dedup
      return [...prev, msg];
    });
    setHasNewMessages(true);
  }

  function onMessageEdited(payload: unknown) {
    const updated = mapPayloadToView(payload);
    setMessages(prev => prev.map(m => m.id === updated.id ? { ...m, ...updated } : m));
  }

  function onReconnect() {
    socket.emit("joinRoom", { roomId });
    scheduleReconnectRefetch();   // debounced 500ms, see Pattern 4
  }

  socket.on("message-created", onMessageCreated);
  socket.on("message-edited", onMessageEdited);
  socket.io.on("reconnect", onReconnect);   // NOTE: socket.io.on, NOT socket.on

  return () => {
    socket.emit("leaveRoom", { roomId });
    socket.off("message-created", onMessageCreated);
    socket.off("message-edited", onMessageEdited);
    socket.io.off("reconnect", onReconnect);
  };
}, [socket, roomId]);
```

**Payload mapping** — the server emits snake_case fields (verified `messages.gateway.ts` lines 203–218). The same `mapMessageView` function already exists in `api.ts` lines 587–608. Import it or duplicate locally. The `message-created` payload shape from `MessagesGateway.broadcastMessageCreated` (lines 197–218):
```
{ conversation_type, conversation_id, message: { id, author_id, author_username, content, reply_to_id, reply_preview, edited_at, conversation_watermark, created_at } }
```

**Reconnect refetch pattern** (`RoomChatView.tsx` — add ref + timer):
```typescript
const reconnectTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

function scheduleReconnectRefetch() {
  if (reconnectTimerRef.current) clearTimeout(reconnectTimerRef.current);
  reconnectTimerRef.current = setTimeout(async () => {
    try {
      const result = await getRoomHistory(roomId);  // last page, no cursor
      setMessages(prev => {
        const existingIds = new Set(prev.map(m => m.id));
        const newMsgs = result.messages.filter(m => !existingIds.has(m.id));
        return newMsgs.length > 0
          ? [...prev, ...newMsgs].sort((a, b) => a.conversationWatermark - b.conversationWatermark)
          : prev;
      });
    } catch { /* silent */ }
  }, 500);
}
```

**Existing send handler pattern** (`RoomChatView.tsx` lines 94–106) — no change needed; REST owns sends.

const scrollRef = useRef<HTMLDivElement>(null);
const [isScrolledUp, setIsScrolledUp] = useState(false);

function handleScroll() {
  const el = scrollRef.current;
  if (!el) return;
  const distFromBottom = el.scrollHeight - el.scrollTop - el.clientHeight;
  setIsScrolledUp(distFromBottom > 100);  // 100px threshold
}

// Auto-scroll only when not scrolled up
useEffect(() => {

    scrollRef.current?.scrollTo({ top: scrollRef.current.scrollHeight });

}, [messages.length, isScrolledUp]);

## Shared Patterns

### useEffect Cleanup (applies to all new useEffect blocks)

**Source:** `apps/web/src/features/messages/DmChatView.tsx` lines 79–129

All new `useEffect` blocks that set up subscriptions or timers must return a cleanup function:

```typescript
// Pattern from DmChatView openConversation effect (lines 87-128)
let cancelled = false;
// ... async work
return () => { cancelled = true; };

// Pattern for WS subscriptions (all chat views):
return () => {
  socket.emit("leaveRoom", { roomId });     // or leaveDm
  socket.off("message-created", onMessageCreated);
  socket.off("message-edited", onMessageEdited);
  socket.io.off("reconnect", onReconnect);  // NOTE: socket.io not socket
};
```

### Named Handler Reference (applies to all socket.on calls)

**Source:** RESEARCH.md Code Examples section

Anonymous functions cannot be removed with `.off()`. Always use named functions:

```typescript
// Correct — named function reference
function onMessageCreated(payload: unknown) { /* ... */ }
socket.on("message-created", onMessageCreated);
// cleanup:
socket.off("message-created", onMessageCreated);

// Wrong — cannot be selectively removed
socket.on("message-created", (payload) => { /* ... */ });
```

### Manager vs Socket Event Target (applies to reconnect handler)

**Source:** RESEARCH.md Pitfall 2

Reconnect events belong to the Manager (`socket.io`), not the Socket:

```typescript
// Correct:
socket.io.on("reconnect", onReconnect);
socket.io.off("reconnect", onReconnect);

// Wrong — handler never fires:
socket.on("reconnect", onReconnect);
```

### Dedup Guard (applies to onMessageCreated handlers)

**Source:** RESEARCH.md Pattern 3

Reconnect may replay events. Always dedup by message ID:

```typescript
setMessages(prev => {
  if (prev.some(m => m.id === newMsg.id)) return prev;
  return [...prev, newMsg];
});
```

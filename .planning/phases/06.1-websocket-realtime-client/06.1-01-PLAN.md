socket.io-client package in the web app. Both actions are required before any
WebSocket integration code can be tested in the browser.

  <action>
In both gateway files change the @WebSocketGateway decorator from:
  `@WebSocketGateway({ cors: { origin: '*' } })`
to:
  `@WebSocketGateway({ cors: { origin: 'http://localhost:4173', credentials: true } })`

Do NOT change anything else in either file. The CORS fix is a one-line change per file.

Rationale (per RESEARCH.md Critical Finding): When socket.io-client sends
`withCredentials: true`, the browser requires a specific `Access-Control-Allow-Origin`
header. The wildcard `*` is blocked by the browser's CORS policy in this case.
The session cookie ("session") is passed in the WS handshake; without this fix
the handshake is rejected before the cookie reaches ws-auth.ts.
  </action>

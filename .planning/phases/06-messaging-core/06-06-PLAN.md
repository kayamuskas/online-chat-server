  const result = await this.db.query<MessageViewRow>(

       m.author_id,
       a.username          AS author_username,
       m.content,
       m.reply_to_id,
       m.edited_at,
       m.conversation_watermark,
       m.created_at,
       rm.author_id        AS reply_author_id,
       ru.username         AS reply_author_username,

     INNER JOIN users a ON a.id = m.author_id
     LEFT  JOIN messages rm ON rm.id = m.reply_to_id
     LEFT  JOIN users ru ON ru.id = rm.author_id

**Step A — ContactsRepository: add `unfreezeConversation`**

Add a new method `unfreezeConversation(userAId: string, userBId: string): Promise<void>` immediately after `freezeDmConversation`. The method normalizes the pair (same `userAId < userBId` pattern) and issues an UPDATE. If no row exists yet, the UPDATE is a no-op — that is correct because if no conversation row exists, there is nothing to unfreeze.

```typescript
async unfreezeConversation(userAId: string, userBId: string): Promise<void> {
  const [a, b] = userAId < userBId ? [userAId, userBId] : [userBId, userAId];
  await this.db.query(
    `UPDATE dm_conversations SET frozen = FALSE WHERE user_a_id = $1 AND user_b_id = $2`,
    [a, b],
  );
}
```

Note: `unfreezeConversation` is called from `unbanUser` which does NOT use a transaction client, so no `SqlExecutor` overload is needed. Keep the signature simple.

async unbanUser(callerId: string, targetId: string): Promise<void> {

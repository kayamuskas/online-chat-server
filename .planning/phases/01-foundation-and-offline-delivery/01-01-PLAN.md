## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer checkout -> workspace build | Untrusted local clone contents and missing vendor assets can break deterministic startup |
| repo manifests -> Docker build context | Package manifests control whether installs stay offline or attempt registry access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I/D | workspace manifests and docs | mitigate | Encode repo-local `vendor/pnpm-store` path and offline requirement in manifests/docs so hidden runtime internet dependency is blocked by design. |
| T-01-02 | T/D | shared queue contract | mitigate | Centralize Redis/BullMQ queue names and connection config in `packages/shared` so later plans do not hand-roll incompatible worker connection policy. |
| T-01-03 | T | future service config | mitigate | Define one shared runtime contract and ports now to prevent later Compose/API/web drift that would create startup races or mismatched endpoints. |
| T-01-13 | T/I | vendored offline dependency store | mitigate | Require `docs/offline-runtime.md` to define a refresh-and-verify procedure tied to `pnpm-lock.yaml` so maintainers have an explicit integrity check before shipping updated vendored dependencies. |
</threat_model>

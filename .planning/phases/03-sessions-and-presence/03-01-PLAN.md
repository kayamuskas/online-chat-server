<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/request headers -> session metadata | IP and UA fields originate outside the server and must be normalized safely |
| authenticated user -> session inventory API | The user must only see and revoke their own sessions |
| revoke operations -> durable session rows | Per-session revoke must not accidentally invalidate unrelated sessions or accounts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | S | client IP metadata | mitigate | Centralize trusted IP extraction with `X-Forwarded-For` fallback rules and explicit normalization. |
| T-03-02 | T/I | session metadata schema | mitigate | Persist browser/IP metadata at session creation in durable rows with tests and explicit schema ownership. |
| T-03-03 | E | session inventory endpoint | mitigate | Scope all inventory queries to the authenticated user and test for cross-user isolation. |
| T-03-04 | T | revoke operations | mitigate | Keep revoke-one and revoke-all-other-sessions as explicit row-level operations with targeted tests. |
</threat_model>
